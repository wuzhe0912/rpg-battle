# V1 — 義大利麵式程式碼

## 這個版本是什麼？

一個完整的 RPG 打怪遊戲，所有程式碼都塞在 `src/App.vue` 一個檔案裡。
功能完整：攻擊、防禦、喝藥水、武器切換、打怪掉寶。

## 檔案結構

```
src/
└── App.vue          ← ~750 行，所有邏輯 + UI + 樣式
```

## 教學重點：為什麼這樣寫有問題？

### 1. 找東西很痛苦

玩家狀態、怪物狀態、遊戲邏輯、UI、CSS 全部混在同一個檔案。
要改怪物攻擊邏輯？在 750 行裡面 Ctrl+F。

### 2. 武器系統的 if/else 地獄

`calcPlayerDamage()` 用一整串 if/else 判斷武器類型。
每加一種武器就要再加一個 else if，而且到處都有類似的判斷邏輯。

### 3. 重複邏輯

HP bar 的顏色計算、武器名稱顯示，在不同地方重複出現。
改一處忘改另一處就出 bug。

### 4. 改一處壞全部

所有東西共用同一個 scope，改玩家邏輯可能不小心動到怪物邏輯。

## 觀眾帶走什麼？

看完 V1，觀眾應該能**感受到痛點**——「這我也寫過」的共鳴感。
這是重構動機的建立，不是批評，而是讓大家理解「為什麼需要拆」。
